# HQ Account - 전표입력 프로그램 PRD (Product Requirements Document)

> 버전: v1.1 | 최종 수정일: 2026-04-01

## 1. 프로젝트 개요

### 1.1 목적
전표를 입력하면 계정과목별 월별 지출 내역표를 자동으로 생성하고 CSV로 출력하는 프로그램

### 1.2 참고 문서
- `2025년재무제표.pdf` — 비용 계정과목 목록 기준
- `2026 월별 지출 내역_v0.1.pdf` — 출력물 형태 기준

### 1.3 배포 형태
- 단일 HTML 파일 (별도 설치 불필요)
- 브라우저에서 바로 실행 (Chrome, Edge 등)
- 다른 PC에 파일 복사만으로 배포 가능

---

## 2. 개발 단계

| 단계 | 범위 | 설명 |
|------|------|------|
| **1단계 (현재)** | 로컬 단독 사용 | 한 사람이 로컬 환경에서 입력/조회/출력 |
| **2단계 (고도화)** | 네트워크 다중 사용자 | 입력자와 조회자가 네트워크로 접근, NocoDB 백엔드 |

| 구분 | 기술 |
|------|------|
| 프론트엔드 | 단일 HTML + CSS + JavaScript |
| 데이터 저장 | localStorage (브라우저 내장) |
| 출력 | CSV 다운로드 |

## 4. 화면 구성

### 4.1 전표입력 (InputPage)

#### 4.1.1 전표 입력 폼
- **날짜**: DatePicker (기본값: 오늘)
- **계정과목**: 드롭다운 (활성 계정과목만 표시, 분류별 그룹)
- **금액**: 숫자 입력 (천 단위 콤마 자동 표시)
- **적요**: 텍스트 입력 (선택)
- 버튼: [저장] [초기화]
- 저장 시 localStorage에 반영 + 폼 초기화 + 하단 목록 새로고침
- 수정 모드: 기존 전표 수정 시 폼에 데이터 채움

#### 4.1.2 CSV 일괄 입력
- CSV 파일 선택 후 **미리보기** 표시
- 각 행의 유효성 검증 (날짜 형식, 계정과목 존재 여부, 금액)
  - 정상: 초록 배지 (OK)
  - 오류: 빨간 배지 (오류 내용 표시)
- [확인 등록]: 정상 데이터만 일괄 저장, 오류 건은 건너뜀
- [양식 다운로드]: CSV 템플릿 다운로드
- CSV 형식: `날짜,계정과목,금액,적요` (첫 행 헤더, UTF-8)

#### 4.1.3 오늘 입력한 전표 목록
- 오늘 날짜 기준 필터링
- 컬럼: No | 날짜 | 계정과목 | 금액 | 적요 | 관리(수정/삭제)
- 하단 요약: 건수, 합계 금액
- 삭제 시 확인 모달 표시

### 4.2 전표조회 (SearchPage)

#### 4.2.1 검색 필터
- **기간**: 시작일 ~ 종료일 (기본값: 당월 1일 ~ 오늘)
- **계정과목**: 드롭다운 (전체 선택 가능)
- 버튼: [검색] [초기화]

#### 4.2.2 검색 결과 테이블
- 컬럼: No | 날짜 | 계정과목 | 금액 | 적요 | 관리(수정/삭제)
- 날짜순 정렬
- 하단 요약: 검색 건수, 합계 금액

### 4.3 월별 지출 내역표 (ReportPage)

#### 4.3.1 연도 선택
- 드롭다운 (현재 연도 ~ 5년 전)
- 버튼: [조회] [CSV 다운로드]

#### 4.3.2 월별 지출 내역표

| 계정과목 | 1월 | 2월 | ... | 12월 | 합계 |
|----------|-----|-----|-----|------|------|
| 직원급여 | 3,500,000 | 3,500,000 | ... | - | 7,000,000 |
| ... | | | | | |
| **합 계** | **금액** | **금액** | ... | **금액** | **총합계** |

- 행: 계정과목별 (계정과목관리 등록 순서)
- 열: 1월 ~ 12월 + 합계
- 금액 없는 셀: `-` 표시
- 하단 합계 행: 월별 전체 합계

#### 4.3.3 CSV 출력
- BOM 포함 UTF-8 CSV (엑셀 한글 호환)
- 파일명: `{연도} 월별 지출 내역.csv`

### 4.4 계정과목관리 (AccountsPage)

#### 4.4.1 계정과목 등록/수정 폼
- **분류**: 드롭다운 (수익 / 판매비와관리비 / 영업외수익 / 영업외비용 / 세금)
- **계정과목명**: 텍스트 입력 (중복 불가)
- **비고**: 텍스트 입력 (선택)
- 버튼: [저장] [초기화]
- 수정 시: 계정과목명 변경되면 기존 전표의 계정과목도 자동 업데이트

#### 4.4.2 계정과목 목록
- 체크박스: [사용안함 항목 표시]
- 컬럼: No | 분류 | 계정과목명 | 비고 | 상태 | 관리(수정/사용안함 토글)
- 분류 순서대로 정렬
- 상태 배지: 사용(초록) / 사용안함(빨강)
- **사용안함** 처리: 삭제 대신 비활성화 → 전표입력/조회 드롭다운에서 숨김
- 비활성 행: 반투명(opacity 0.5) 표시

## 6. 공통 사항

### 6.1 네비게이션
- 좌측 사이드바 (고정 220px)
- 메뉴: 전표입력 | 전표조회 | 월별내역표 | 계정과목관리

### 6.2 금액 표시
- 천 단위 콤마: `1,000,000`
- 원 단위 (소수점 없음)

### 6.3 날짜 형식
- 표시/저장: `YYYY-MM-DD`

### 6.4 알림
- Toast 메시지 (우측 하단, 2초 후 자동 사라짐)

### 6.5 삭제 확인
- 모달 팝업으로 확인 후 삭제

---

## 7. 향후 고도화 사항 (2단계)

- NocoDB 백엔드 연동 (React + TypeScript 전환)
- 네트워크 접근 (다중 사용자)
- 사용자 역할 구분 (입력자 / 조회자)
- 사용자 인증/권한 관리
- PDF 출력 기능 (jsPDF)
- 예산 금액 입력 및 실적 대비 예산 비교
- 전표 승인 워크플로우
- 데이터 내보내기/가져오기 (PC 간 이동)
